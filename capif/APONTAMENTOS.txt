O projeto estava configurado para usar imagens Docker armazenadas num registry privado da ETSI, ao qual eu não tinha acesso. Por isso alterei os Dockerfiles para utilizar imagens oficiais públicas do Docker Hub com a mesma funcionalidade. Também ajustei o nginx.conf porque uma configuração específica da ETSI não era compatível com a versão oficial do Nginx. Estas alterações permitiram executar o OpenCAPIF localmente sem alterar a lógica da aplicação



registry -> servidor onde ficam guardadas as imagens Docker

container -> a app a correr

Uma imagem Docker é um ficheiro que contém tudo o que uma aplicação precisa para funcionar:

    Sistema operativo base
    Python ou outra linguagem
    Bibliotecas
    Configurações
    Código da aplicação


O registry ETSI é público para leitura (pull). Só não consegues fazer push (enviar imagens). As tuas alterações nos Dockerfiles foram para quando quiseres fazer build localmente — se usares as imagens pré-construídas deles (que é o que acontece por omissão), não precisas das tuas alterações.

cosnigo dar pull deles mas nao da para dar push

O Register é um serviço auxiliar simples que só faz uma coisa: verificar se o utilizador existe na BD e dar-lhe um passe temporário (JWT) para entrar no CAPIF Core.

O CAPIF Core é o sistema real — gere APIs, Providers, Invokers, faz tudo.
